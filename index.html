错题集

Python中的深拷贝与浅拷贝

错误率：68%

题目：

以下代码执行后，`list_b` 的值是什么？ 

list_a = [1, [2, 3], 4]
list_b = list_a.copy()
list_a[1][0] = 99
print(list_b)

常见错误答案：

[1, [2, 3], 4]

正确答案：

[1, [99, 3], 4]

解析：

使用 list.copy() 进行的是浅拷贝，只会复制列表的外层结构，而内部的嵌套对象仍然是引用。因此，修改 list_a[1][0] 会影响到 list_b 中的对应元素。

Python中的闭包陷阱

错误率：72%

题目：

以下代码执行后，`functions[0]()` 的输出是什么？ 

def create_functions():
    functions = []
    for i in range(3):
        functions.append(lambda: i)
    return functions

functions = create_functions()
print(functions[0]())

常见错误答案：

0

正确答案：

2

解析：

闭包会捕获变量的引用而不是值。当循环结束时，`i` 的值为2，因此所有闭包都引用同一个值2。正确的做法是使用默认参数来绑定当前的值。

Python中的异常处理

错误率：55%

题目：

以下代码会抛出什么异常？ 

def divide(a, b):
    try:
        result = a / b
    except ZeroDivisionError:
        print("除数不能为零！")
    else:
        return result
    finally:
        print("执行完毕")

divide(5, 0)

常见错误答案：

ZeroDivisionError

正确答案：

没有异常抛出，程序会打印： 

除数不能为零！
执行完毕

解析：

代码中已经捕获了 ZeroDivisionError 异常，因此不会向上抛出。`finally` 子句中的代码无论是否发生异常都会执行。

Python中的生成器

错误率：61%

题目：

以下代码的输出是什么？ 

def generate_numbers():
    for i in range(3):
        yield i

gen = generate_numbers()
print(list(gen))
print(list(gen))

常见错误答案：

[0, 1, 2]
[0, 1, 2]

正确答案：

[0, 1, 2]
[]

解析：

生成器只能遍历一次。第一次调用 list(gen) 时，生成器会生成所有值。再次调用时，生成器已经耗尽，不会产生任何值。
